### Задание 1

На лекции рассматривались режимы репликации master-slave, master-master, опишите их различия.

Ответить в свободной форме

 ***Master-Slave***

 В этой архитектуре есть один главный сервер (Master) и один или несколько  серверов Slave. Только Master принимает запросы на запись (CREATE, UPDATE, DELETE),изменения записываются в лог и  передаются на серверы Slave, которые могут только читать.

 Схема актуальна,если имеется много запросов на чтение,которые можно распределить по нескольким Slave серверам, сняв нагрузку с главного. 

 **Из минусов:** если Master упадет,система не сможет принимать новые данные, пока один из Slave не будет вручную или автоматически повышен до уровня главного.

***Master-Master***


Любой из серверов  может принимать как запросы на чтение, так и запросы на запись. Пользователи могут работать с ближайшим сервером, уменьшая задержки,нет единственного узла записи из-за  устойчивой системы  к сбоям.

К **минусам** можно отнести сложность обеспечения согласованности.Если два сервера одновременно изменяют одни и те же данные, возникает конфликт,требующий механизма разрешения.
